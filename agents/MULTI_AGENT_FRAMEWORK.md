## 🔧 技术实现

### 1. Agent 配置

每个 Agent 在 `agents/<name>/` 目录下包含：
- `TASKS.md` - 任务清单
- `context.md` - 背景信息
- `skills/` - 技能配置

### 2. 通信机制

- 所有 Agent 读取共享的 `PROJECT_PLAN.md`
- 每日更新 `TASK_STATUS.md`
- 通过文件系统的 `/workspace/memory/` 传递信息

### 3. 任务流转

```
新任务 → John 分配 → Agent 执行 → 更新状态 → 完成
```

## 🔄 工作流程

### 每日流程

1. **晨会**（自动）：检查任务状态
2. **执行**：Agent 按优先级执行任务
3. **更新**：完成后更新状态
4. **汇报**：每日 18:00 生成进度报告

### 任务分配

- 高优先级：立即执行
- 中优先级：当日完成
- 低优先级：本周完成
