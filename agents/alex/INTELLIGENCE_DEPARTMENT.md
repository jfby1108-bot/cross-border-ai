## 📅 任务时间表

### 每日任务（3次 × 10分钟 = 30分钟/天）

| 时间 | 任务 | 执行者 | 时长 |
|------|------|--------|------|
| **06:00** | 早间扫描：竞品动态 + 新工具 | Iris (Cron 触发) | 10 分钟 |
| **14:00** | 午间扫描：竞品动态 + 新工具 | Iris (Cron 触发) | 10 分钟 |
| **22:00** | 晚间扫描：竞品动态 + 新工具 | Iris (Cron 触发) | 10 分钟 |

**扫描内容**：
1. 检查竞品官网/社交媒体更新
2. 搜索 Product Hunt trending
3. 检查 GitHub AI/automation trending
4. 更新 `intelligence/new-tools.md`

### 每 3 天任务（18:00）

| 时间 | 任务 | 执行者 | 时长 |
|------|------|--------|------|
| **18:00** | 深度分析：竞品功能 + 工具评估 | Luna (Cron 触发) | 1 小时 |

**分析内容**：
1. 汇总最近 3 天竞品动态
2. 评估新工具集成可能性（API、成本、收益）
3. 撰写 `intelligence/deep-analysis.md`

### 每周日任务（18:00）

| 时间 | 任务 | 执行者 | 时长 |
|------|------|--------|------|
| **18:00** | 周度趋势报告 | Luna (Cron 触发) | 1 小时 |

**报告内容**：
1. 本周竞品动态汇总
2. 新工具/技能发现总结
3. 行业趋势分析
4. 撰写 `intelligence/trends-weekly.md`
5. 制定下周计划

## 📋 日常工作流程

### 扫描流程（Iris）
```
1. Cron 触发 (06:00/14:00/22:00)
2. 检查竞品更新 → 记录到 live-updates.md
3. 搜索新工具 → 记录到 new-tools.md
4. 汇总报告 → 发送通知
```

### 分析流程（Luna）
```
1. Cron 触发 (每3天 18:00 / 每周日 18:00)
2. 读取近期扫描数据
3. 深度分析 + 评估
4. 生成报告
5. Alex 审核
```

### 决策流程（Alex）
```
1. 收到分析报告
2. 评估集成建议
3. 决定是否加入开发计划
4. 更新优先級
```
